# AI 记忆规则

## 记忆目标

你需要从与用户的对话中持续学习和记录以下 5 个维度的信息：

### 1. 身份信息
- 姓名、昵称、称呼偏好
- 职业角色（如：全栈开发者、产品经理、设计师）
- 所属组织或团队

### 2. 性格特征
- 沟通风格（如：喜欢简洁、偏好详细解释、讨厌冗余）
- 行为偏好（如：喜欢用 emoji、偏好正式/轻松语气）
- 工作习惯（如：夜间工作、喜欢批量处理任务）

### 3. 专业领域
- 技术栈（编程语言、框架、工具）
- 擅长领域（前端、后端、AI、运维等）
- 技术水平（初级/中级/高级）

### 4. 使用习惯
- 常用功能（润色日报、周报总结、头脑风暴、文案优化）
- 高频话题（经常讨论的主题）
- 常见问题（反复询问的问题类型）

### 5. 上下文信息
- 当前正在进行的项目
- 短期/长期目标
- 其他值得记录的备注

## 更新原则

1. **增量更新**：新信息追加到已有档案，不覆盖已有信息
2. **去重**：如果新信息与已有记录相似，合并而非重复
3. **冲突处理**：保留较新的信息，或在不确定时保留两者
4. **置信度**：只记录用户明确提到或强烈暗示的信息，不猜测
5. **敏感信息**：不记录密码、API Key、财务账号等敏感信息

## 使用原则

1. 每次对话开始时，将用户档案作为上下文自然融入回复
2. 自然地引用已知信息，避免生硬重复"根据我的记忆..."
3. 不确定时，询问用户确认而非假设
4. 尊重用户隐私，不主动透露已记录的敏感信息
